The Loop

Ruby vs. Javascript

November 15, 2015

You'll soon find out that one of the most effective ways of manipulating objects in an array is through a loop. However, instead of thinking, I'm going to change this entire group or array, looping can effect certain individual elemnts in an array, depending on how your code is written. JavaScript and Ruby both have different ways of dealing with loops.

Looping with JavaScript

JavaScript is able to utilize a single line of code to set the conditions of a loop. 


array = [1, 2, 3, 4, 5]

for( var i = 0; i < array.length; i ++){
  console.log(array[i]*2);
}

//output => 2, 4, 6, 8, 10

The above code will take all numbers in array and multiply them by two. If you look closely, there are three sections within the parenthesis that follow for. The first section var i = 0 sets a variable for the loop. For the purpose of this loop, and many loops in JavaScript, the variable created will act as an index point for the array we are dealing with. The second section let's us know when the loop will end. The loop tells us, as long as this condition is true, var i is less than the array's length, we will execute code below. The third section i++ tells us that var i will increase by one each time console.log is executed.

This is the main way to loop over each item in an array by using JavaScript. You can do the same thing in Ruby, but the syntax looks a little different. Let's look at how to achieve the same output using Ruby.

Looping in Ruby

array = [1, 2, 3, 4, 5]

array.each do |i|
  print i * 2
end

//output => 2, 4, 6, 8, 10

The code above looks much simpler. That is because Ruby has some built in methods that can manipulate objects more precisely. Four our Ruby loop, we don't need to define index positions, but we can just use the .each method to iterate through each item in an array. For the code above, each element in the array will receive the temporary placeholder i according to the iteration. Next, we will print that item multiplied by two.

In general, working with Ruby is much easier. Ruby was created to be easier on the user. Thus, in Ruby we are able to receive the same output using less code. Ruby has a library of built-in methods at our disposal which can be used to cut back on the amount of code we need to write.
