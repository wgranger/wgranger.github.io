Working Through Ruby Classes

November 8, 2015

Once you've started working through some basic methods and become familiar with strings, integers, and arrays in Ruby, you may want to create a class of your own. Just how Ruby endows particular methods to a "string" class or an "array" class, you too can create a class of your own with custom methods that you define.

Why use a class?

The question is valid. Why use a class rather than using a method in Ruby? Using classes will save a good bit of time and frustration in Ruby in that we can append objects with a method to simplify our code rather than rewriting methods. If we wanted to keep track of a record collection using Ruby, we could simply create a class of Records and then create our own methods that could return a release date, artist, record label, or whatever we wanted to know. This would be as easy as using instance variables and methods with our objects. 


class Records

end

Creating a class is just as easy as creating a method. The above code will create a class in Ruby. The one thing you have to remember, following Ruby conventions, is to capitalize the name of your class. This is a common way to distinguish class names from other items in Ruby.


class Records
  def initialize(artist, album, year)
    @artist = artist
    @album = album
    @year = year
  end
end

We took one step further with our code above and assigned some instance variables to our class. Instance variables are a bit more powerful than local variables in that they can be used in various methods. If you recall, local variables can only be used in the method they were created. This isn't too helpful with classes in that we'll want to use instance variables in a number of different methods. Simply include the @ sign at the beginning of your variable and you've got a much more powerful instance variable.

Also, note that I used an initialize method that took a number of arguments: artist, album, and year. The initialize method will be performed automatically with each new object that is created of that class. Although unnecessary, the initialize method is often used when creating a new Ruby class. Note how I used three arguments in my initialize method. Although this isn't necessary, I included them to show how I can pass information to a class when creating a new object of the record class. 


class Records
  def initialize(artist, album, year)
    @artist = artist
    @album = album
    @year = year
  end

  def year_first
    p "#{@year}: #{@artist}, #{@album}"
  end

  def album_first
    p "#{@album} by #{@artist} was released in #{@year}."
  end
end

Our class is getting a bit more fleshed out. Here we have a couple instance methods that will put some information into the console. Remember, these instance methods can only be called upon if there is an object to attach it to. In order to create a new object of a particular class, I would have to run the following code.


john = Records.new

I could then use my instance methods to get just the information I was looking for. The following method calls. 


john.year_first
john.album_first

Would provide the output below.


"1959: John Coltrane, Giant Steps"
"Giant Steps by John Coltrane was released in 1959"

This is a quick introduction to creating classes in Ruby and endowing them with certain methods. Classes are good for reusing methods that do the same thing. The last thing you want in your code are redundant methods. Reusing methods will save time, energy, money, and sanity.
